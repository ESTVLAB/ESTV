## Reporting a Vulnerability

The ESTV team takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings.

### How to Report

**DO NOT** create public GitHub issues for security vulnerabilities.

Instead, please report security vulnerabilities by emailing:

**security@estvlabs.com**

Please include:

1. **Description** of the vulnerability
2. **Steps to reproduce** the issue
3. **Potential impact** assessment
4. **Suggested fix** (if any)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Timeline**: Depends on severity
  - Critical: 24-72 hours
  - High: 7 days
  - Medium: 30 days
  - Low: 90 days

### Bug Bounty Program

We are currently developing a bug bounty program. Details will be announced on our official channels.

### Smart Contract Security

Our smart contracts undergo:

- Internal security reviews
- External audits by reputable firms (CertiK, Hacken)
- Continuous monitoring

Audit reports will be published in the `/audits` directory upon completion.

## Security Best Practices

### For Users

1. **Verify Contract Address**: Always confirm you're interacting with the official contract
   ```
   7GovpZ67R8t3NssZWkFE6pKL6HUVTXwkv9C1RTDADRY
   ```

2. **Official Channels Only**: Only use links from our verified sources
   - Website: https://estv-token.io
   - Twitter: @estvesportstv (verified)
   - Telegram: @estv_esportstv

3. **Never Share Private Keys**: ESTV team will NEVER ask for your private keys or seed phrases

4. **Beware of Scams**: Report impersonators to security@estvlabs.com

### For Developers

1. Follow secure coding practices
2. Never commit secrets or private keys
3. Use environment variables for sensitive data
4. Keep dependencies updated
